# 面试讲解

## 项目描述

PromoBrain 是一个面向电商广告推广场景的智能知识库与广告投放模拟平台。系统支持商品资料和广告规则入库、RAG 检索、广告素材生成、关键词多路召回、候选排序、广告计划发布、预算扣减、素材审核和 CTR/CVR/ROI 效果分析。

技术栈包括 Spring Boot 3、Vue3、FastAPI、Redis、RabbitMQ、MySQL、MinIO、Qdrant 和 OpenAI-compatible 大模型接口。

## 负责内容

1. 设计轻量 RBAC 权限系统，支持用户、角色、权限点和商家级数据隔离。
2. 设计商品中心、知识库、广告计划、素材、关键词、预算流水、广告事件日志等核心业务模型。
3. 设计广告关键词多路召回链路，综合商品标题、商品卖点、用户评价、历史高转化关键词、竞品关键词和向量语义召回。
4. 设计广告候选多因子排序策略，综合语义相关性、预估 CTR、预估 CVR、商品毛利和素材风险分。
5. 设计预算扣减高并发方案，基于 Redis Lua 实现预算判断与扣减的原子操作。
6. 通过 requestId 幂等键和 MySQL 唯一索引避免重复扣费，并通过 RabbitMQ 异步落库预算流水。
7. 设计广告请求在线链路，将广告计划、商品、素材和关键词缓存至 Redis。
8. 设计 RAG 与推广 Agent 能力，将商品知识检索、广告规则校验、关键词推荐、文案生成和投放建议编排为工具调用流程。

## 重点讲法

项目里我重点做了两个方向。

第一个是 RAG/Agent 工程化。商品资料和广告规则上传后会被切分、向量化、入库，生成素材或推荐关键词时先召回相关商品卖点、用户评价和平台规则，再让大模型生成结果，避免模型脱离业务知识胡编。

第二个是广告后端链路。广告计划发布后会构建 Redis 缓存，广告请求主链路尽量不查 MySQL，曝光点击日志通过 MQ 异步写入。预算扣减没有用简单查库更新，而是用 Redis Lua 保证预算判断和扣减的原子性，通过 requestId 做幂等，数据库侧用唯一索引兜底。
