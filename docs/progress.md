## 阶段规划

### 第一阶段：基础骨架

目标：

- Spring Boot 后端启动。
- Vue3 前端启动。
- MySQL 初始化。
- Redis 接入。
- Sa-Token 登录。
- 轻量 RBAC。
- 商品 CRUD。
- 广告计划 CRUD。

验收：

- 能登录。
- 能根据角色控制菜单。
- 能新增商品。
- 能新增广告计划。

### 第二阶段：知识库与 AI 服务

目标：

- MinIO 文件上传。
- `knowledge_doc` 表。
- `async_task` 表。
- FastAPI 服务。
- 文档解析。
- chunk 切分。
- mock embedding。
- Qdrant 入库。
- 语义检索接口。

验收：

- 能上传商品资料。
- 能生成 chunk。
- 能检索知识库内容。

### 第三阶段：素材生成与关键词推荐

目标：

- 广告文案生成。
- 广告规则召回。
- 关键词多路召回。
- 候选排序。
- 素材保存。
- 素材审核任务。

验收：

- 能选择商品生成广告文案。
- 能推荐关键词。
- 能看到排序分数。

### 第四阶段：广告在线链路

目标：

- 广告计划发布。
- Redis 缓存广告计划。
- 广告请求模拟。
- 广告候选召回。
- 广告排序。
- 曝光日志 MQ。

验收：

- 输入搜索词能返回广告。
- 曝光日志能异步写入。

### 第五阶段：预算扣减高并发

目标：

- Redis Lua 预算扣减。
- requestId 幂等。
- MQ 异步预算流水。
- MySQL 唯一索引兜底。
- 预算不足处理。
- 重复请求处理。
- 预算对账任务。

验收：

- 高并发点击不会超扣。
- 重复 requestId 不会重复扣费。
- 预算不足返回明确状态。

### 第六阶段：看板与压测

目标：

- CTR/CVR/ROI 看板。
- 预算流水页。
- 任务中心页。
- k6/JMeter 压测脚本。
- README 截图。
- 部署文档。

验收：

- README 可展示完整业务闭环。
- 可以 `docker-compose up -d` 一键启动。
- 有压测说明和结果截图。
